# 原型链

其实原型链本质上也是一个对象，当你声明一个构造函数的时候，就会有一个原型对象

# 图解

万物皆对象

# 三大规则

  • 实例对象通过 _proto_ 指向对应的原型对象
  • 构造函数通过 prototype 指向对应的原型对象
  • 原型对象通过 constructor 指向对应的构造函数或实例对象

# 其他

  • 原型对象通过 _proto_ 指向 Object 的原型对象
  • Object 原型对象通过 constructor 指向 Object 构造函数
  • Object 构造函数通过 prototype 指向 Object 原型对象
  • Object 构造函数通过_proto_ 指向 null
  • Object 构造函数通过实例化指向Function原型对象

# 作用

  1. 可以节约内存空间
  2. 减少变量名污染
  3. 让函数与函数间更加紧密
  4. 主要作用是为了继承

# 三大特性

  1. 封装
  • 好处是可以复用，降低冗余度，更有利于模块化编程，能够为变量函数提供更多的保护
  1. 继承
  • 子类可以获得父类的属性和方法
  1. 多态
  • 对于同一个操作（指令），不同的对象表现出不同的反应、隐藏不同，灵活运用
